1. Project Title & Description

Title: Help Desk Ticket Analytics System

Description: The Help Desk Ticket Analytics System is a project designed to turn raw IT support ticket data into actionable insights. By analyzing and identifying the most frequent recurring issues, the system highlights opportunities to improve processes, reduce workload, and enhance overall support efficiency. This project demonstrates how data-driven decision making can transform everyday IT operations into more proactive and effective services.

4. High-Level Scope

In Scope for Semester Project

  • Collect or simulate help desk ticket data (CSV, SQL, or dataset source).
  • Clean and prepare the data for analysis (Excel, SQL, or Python).
  • Develop an analytical model to identify the top 3–5 recurring IT issues.
  • Create visualizations (Excel, Power BI, or Python) to show patterns and trends.
  • Provide actionable recommendations to reduce recurring issues.
  • Document the system and publish results to a GitHub/portfolio site.
  • Deliver a professional presentation summarizing findings and recommendations.

Roadmap Items (Future Expansion)

  • Build an interactive dashboard in Power BI or Tableau for ongoing monitoring.
  • Explore predictive modeling (e.g., forecasting ticket volume or resolution time).
  • Automate parts of the workflow using Python scripts or APIs.
  • Integrate AI-based classification for categorizing ticket issues.
  • Apply the system to live ticket data at work for real-world validation.

Out of Scope

  • Deploying a production-grade ITSM or enterprise analytics platform.
  • Full automation or AI integration during this semester.
  • Making organizational changes at PCC or implementing solutions live.
  • Managing confidential or sensitive data from real systems without anonymization.

6. Project Methodology Selection

Chosen Methodology: Hybrid–Incremental

Reasons:

  • Each deliverable is a distinct increment that builds toward the final system.
  • Allows flexibility in tools (Excel, Power BI, or Python/SQL) without breaking the plan.
  • Fits the semester timeline by spacing out work in logical stages.
  • Keeps the project structured and professional while avoiding the rigidity of pure Predictive.
